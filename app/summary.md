# 🔍 Why This Tech Stack Was Selected

This project is built using a Retrieval-Augmented Generation (RAG) architecture.  
Below is the rationale behind selecting each core component.

---

## 🧠 Large Language Model (LLM)

### openai/gpt-oss-120b (via Groq)

The `openai/gpt-oss-120b` model was selected for its strong reasoning and text generation capabilities, which are essential for high-quality question answering and document summarization.

**Why this model:**

- High-quality contextual understanding
- Strong performance in multi-turn conversations
- Effective summarization capabilities
- Suitable for long context inputs (RAG-based systems)

Using it via Groq provides:

- ⚡ Extremely low inference latency
- 💰 Cost-efficient performance at scale
- 🚀 Production-ready API performance

This makes it ideal for real-time PDF chat and summarization systems.

---

## 📦 Vector Store

### FAISS (L2 Distance)

FAISS was chosen as the vector database for semantic search.

**Why FAISS:**

- High-performance similarity search
- Optimized for nearest neighbor retrieval
- Lightweight and suitable for local deployment
- Easy to save and load indexes
- Ideal for offline or self-hosted environments

L2 Distance was selected as the similarity metric because it works effectively with dense embedding vectors generated by transformer-based models.

FAISS ensures fast and scalable document retrieval within the RAG pipeline.

---

## 🔎 Embedding Model

### sentence-transformers/all-MiniLM-L6-v2

This embedding model was selected for converting text chunks into dense vector representations.

**Why this model:**

- Lightweight and fast (~80MB)
- High semantic accuracy relative to its size
- Optimized for semantic similarity tasks
- Widely used in RAG and search systems

It provides an excellent balance between performance and computational efficiency.

---

## 🏗️ Why This Combination Works Well Together

| Component | Role | Benefit |
|------------|------|----------|
| MiniLM-L6-v2 | Text → Vector embeddings | Fast and efficient semantic encoding |
| FAISS (L2) | Vector storage & retrieval | High-speed similarity search |
| GPT-OSS-120B | Final answer generation | Strong reasoning and natural responses |

### Resulting Advantages

- ⚡ Fast retrieval
- 🧠 High-quality responses
- 💬 Multi-turn conversation support
- 📄 Accurate PDF summarization
- 🚀 Production-ready architecture
